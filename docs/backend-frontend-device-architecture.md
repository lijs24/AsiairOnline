# 本地后端、网页前端与 ASIAIR 设备交互关系

## 角色边界

| 角色 | 连接对象 | 主要职责 |
| --- | --- | --- |
| ASIAIR 盒子 | 只接受本地服务器后端连接 | 提供 JSON-RPC、图像端口、SMB 素材共享 |
| 本地服务器后端 | 连接 ASIAIR；向浏览器提供 HTTP API | 后台轮询、状态缓存、写入排队、图像拉取、备份任务 |
| 网页前端 | 只连接本地服务器后端 | 展示后端缓存；在获得主控后提交相机写入动作 |
| Tailscale 用户 | 通过 Tailnet 访问本地服务器网页 | 查看同一份后端状态；按设备申请主控 |

## 读写原则

| 类型 | 由谁发起 | 是否直连 ASIAIR | 说明 |
| --- | --- | --- | --- |
| 状态查看 | 浏览器 | 否 | 浏览器读取后端缓存，不因为打开网页或切换设备就直接访问盒子 |
| 后台轮询 | 后端 | 是 | 后端按设备线程周期刷新相机、曝光、温度、存储和图像缓存 |
| 图像查看 | 浏览器 | 否 | 浏览器读取后端保存的 PNG/RAW 缓存；手动拉图由后端访问图像端口 |
| 参数写入 | 浏览器提交，后端执行 | 是 | 只有持有该设备主控的会话可以写入 |
| 快门拍摄 | 浏览器提交，后端执行 | 是 | 写入曝光/Bin、切到预览、下发快门均由后端串行执行 |
| 素材备份 | 后端 | 是 | 通过 SMB/robocopy 增量复制，遵守不删除策略 |

## 多用户规则

| 场景 | 规则 |
| --- | --- |
| 多个用户同时打开网页 | 都读取同一份后端缓存，互不触发额外设备刷新 |
| 用户切换设备 | 前端切换到该设备已有缓存；若缓存尚未就绪，等待后台刷新 |
| 多个用户查看同一设备 | 无冲突，都是只读 |
| 多个用户操作同一设备 | 同一时间只有一个主控会话可以写入 |
| 一个用户操作多个设备 | 主控按设备独立持有，切换设备不自动释放其他设备主控 |

## 多设备与端口调度

| 范围 | 规则 |
| --- | --- |
| 不同 ASIAIR 盒子 | 互不阻塞，可以并行轮询、拉图或执行操作 |
| 同一盒子同一端口 | 串行访问，避免两个请求同时占用同一个 ASIAIR 服务端口 |
| 同一盒子不同端口 | 平时可以并行，例如 RPC 4700 与图像 4800 可各自工作 |
| 同一盒子关键写入窗口 | 相机写入、曝光设置、快门等高优先级动作会阻止低优先级轮询和图像拉取插队 |
| 后台刷新 | 低优先级，遇到写入动作会等待或快速放弃本轮，保留旧缓存 |

## 当前写入优先级

| 优先级 | 操作 |
| --- | --- |
| 第一优先级 | `set_camera_exp_and_bin`、`set_control_value`、`set_page`、`start_exposure`、写入后的读回确认 |
| 第二优先级 | 相机状态、曝光参数、控制项、16 Bit 状态、子画幅、缓存图像信息 |
| 第三优先级 | 手动拉取当前图像、后台图像缓存 |
| 第四优先级 | 素材容量扫描、备份状态展示、日志读取 |

## 缓存策略

| 缓存 | 位置 | 用途 |
| --- | --- | --- |
| 相机状态缓存 | 内存 + `state/camera-cache/*.json` | 网页打开和切换设备时直接展示 |
| 图像预览缓存 | `state/image-preview/<device>/current.png` 与 `current.raw16be` | 网页图像预览、STF 拉伸、本地 RAW 拉伸 |
| 备份状态 | `state/latest.json` 与 `state/runs/` | 总览页展示备份进度和历史 |
| 日志 | `logs/` | robocopy 与运行记录 |

## 失败处理

| 情况 | 处理 |
| --- | --- |
| 盒子离线 | 保留最后一次有效缓存，并标记本轮刷新为部分失败 |
| 后台轮询超时 | 不覆盖已有的非空状态字段 |
| 写入 RPC 超时 | 立即读回确认；读回匹配则视为成功并记录警告 |
| 写入读回不匹配 | 返回失败，前端保留用户输入，不用后台旧值覆盖正在编辑的表单 |
| 图像端口返回 8 Bit 或 16 Bit | 后端按实际 `raw_data` 大小识别，不再用 Bin 推断字节数 |
